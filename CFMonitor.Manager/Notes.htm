CFMonitor

This application enables various items to be monitored automatically and to take a particular action depending on the outcome. For example, Test a particular URL and send an email to the Technical Support team if the URL cannot be opened due to loss of connectivity to the web server.

Items that can be monitored

Item monitored What can trigger an action
DHCP
Disk space
DNS
File
  • File exists
  • File does not exist
  • File contains specific piece of text
  • File does not contain specific piece of text
LDAP
Memory
Registry
REST (Web service)
  • Particular HTTP status
  • Particular lower level error status
  • Response content contains a particular value
SMTP Server
SOAP (Web service)
  • Particular HTTP status
  • Particular lower level error status
  • Response content contains a particular value
Socket (TCP/UDP)
  • Socket connected
  • Socket not connected
Server ping
  • Ping success
  • Ping failed
Windows service
  • Service running
  • Service not running
URL
  • Particular HTTP status
  • Particular lower level error status
  • Response content contains a particular value
Process
  • Process running
  • Process not running
SQL query
  • Query returns rows
  • Query returns no rows
  • Exception running query

Actions that can be taken

Action Comments Examples
Send email If action is triggered by running SQL query then all resultsets returned by the query will be included as CSV attachments. Send an email to the Technical Support desk.
Open URL Call a web service.
Run SQL query. Insert record in to table that is picked up and displayed in a UI somewhere
Run a particular process.
